# Правила проекта MAX

- Исходный файл не должен превышать 1000 строк.
- При приближении к 800 строкам логику нужно заранее разделить по ответственности.
- Превышение 1000 строк допустимо только в исключительном случае, когда разделение
  объективно ухудшает связность; причина должна быть явно записана в `PLAN.md`.
- Крупные справочные данные разделяются по логическим категориям и не смешиваются
  с исполняемым кодом.
- PostgreSQL является источником истины; frontend-файлы могут содержать только
  кэш, адаптеры и начальную загрузку, но не рабочую копию каталога.
